Manages a Lightdash space within a project. Spaces are used to organize charts and dashboards. This resource allows you to create, update, and delete spaces, and manage their visibility (public or private). You must specify the project UUID where the space will reside and a name for the space. Nested spaces are supported by specifying a parent space UUID. Both root and nested spaces can have their own access control settings (Restricted Access).

The `is_private` argument maps to the Lightdash API `inheritParentPermissions` flag on create and update: public spaces (`is_private = false`) set `inheritParentPermissions` to `true` so the space inherits project permissions; private / restricted spaces (`is_private = true`) set it to `false`. The provider reads the same semantics back from the API when refreshing state.
